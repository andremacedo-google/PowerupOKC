# Acervo de Dados Não Estruturados (Bases de Conhecimento)

Este arquivo de `index.md` atua como o catálogo de navegação unificado e plano para o diretório `/data-objects/unstructured/` da **PowerUp OKC**, estruturado em conformidade com as regras de progressive disclosure do padrão **Open Knowledge Format (OKF) v0.1**.

Ele lista as **23 bases de dados não estruturados, acervos jurídicos, e guias normativos** do grupo, mapeadas em uma única lista plana sem divisões físicas de subpastas, sob o mesmo diretório raiz `/data-objects/unstructured/`. Esta arquitetura simplificada atua como a fundação semântica indispensável para o treinamento e operação síncrona de agentes cognitivos baseados em **Retrieval-Augmented Generation (RAG)** sem riscos de alucinações técnicas.

## 2. Princípios de Governança de Informações Não Estruturadas (LeanIX v4)

De acordo com as diretrizes do metamodelo da **SAP LeanIX v4** e as melhores práticas do setor de utilidades elétricas:

1.  **Abstração de Documentos:** O LeanIX não é uma ferramenta de gerenciamento eletrônico de documentos (GED) ou um repositório de arquivos de trabalho individuais. Os artefatos não estruturados são modelados como Fact Sheets lógicos de 'Data Object' representativos (ex: modelar 'PRODIST', não o PDF físico 'modulo_3_v4.pdf').
2.  **Mapeamento de SOT (System of Truth):** Cada acervo documental permanente possui um único sistema proprietário canônico encarregado de sua custódia e controle (ex: o SharePoint Jurídico para os PPAs, ou o Confluence para os TDDs de TI/TO).
3.  **Linhagem e Análise de Impacto:** Conectar as bases não estruturadas a aplicações e equipes permite analisar o impacto em cascata de indisponibilidades lógicas de rede (ex: se o repositório de runbooks estiver inoperante, quais processos operacionais do COD param).
4.  **Celeridade Operacional com IA:** As bases de conhecimento documentais homologadas atuam como os repositórios semânticos de grounding que alimentam síncronamente os 71 agentes de IA do portfólio.
